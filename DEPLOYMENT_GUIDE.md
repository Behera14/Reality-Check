# 🚀 DeepFake Detection App - Render Deployment Guide

## 📋 Prerequisites

1. **GitHub/GitLab Account**: Your code should be in a Git repository
2. **Render Account**: Sign up at [render.com](https://render.com)
3. **Model Access**: Ensure your Hugging Face model is accessible

## 🔧 Deployment Steps

### Step 1: Push Code to Repository

```bash
# If using GitLab
git push gitlab main

# If using GitHub (you'll need proper permissions)
git push origin main
```

### Step 2: Deploy on Render

1. **Go to Render Dashboard**
   - Visit [dashboard.render.com](https://dashboard.render.com)
   - Click "New +" → "Web Service"

2. **Connect Repository**
   - Choose your Git provider (GitHub/GitLab)
   - Select your repository: `dfd` or `gettechthings-project`

3. **Configure Service**
   - **Name**: `deepfake-detection-app`
   - **Environment**: `Python 3`
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `python server.py`

4. **Environment Variables**
   ```
   FLASK_ENV=production
   SECRET_KEY=<auto-generated>
   PORT=10000
   MEDIAPIPE_DISABLE_GPU=1
   KMP_DUPLICATE_LIB_OK=True
   ```

5. **Advanced Settings**
   - **Health Check Path**: `/`
   - **Auto-Deploy**: Enabled
   - **Plan**: Free (or upgrade for better performance)

### Step 3: Alternative - Use render.yaml

If you prefer configuration as code, your `render.yaml` is already set up:

```yaml
services:
  - type: web
    name: deepfake-detection-app
    env: python
    plan: free
    buildCommand: pip install -r requirements.txt
    startCommand: python server.py
    envVars:
      - key: FLASK_ENV
        value: production
      - key: SECRET_KEY
        generateValue: true
      - key: PORT
        value: 10000
      - key: MEDIAPIPE_DISABLE_GPU
        value: "1"
      - key: KMP_DUPLICATE_LIB_OK
        value: "True"
    healthCheckPath: /
    autoDeploy: true
```

## 🔍 Troubleshooting

### Common Issues:

1. **Build Failures**
   - Check `requirements.txt` for version conflicts
   - Ensure all dependencies are compatible

2. **Model Loading Issues**
   - Verify Hugging Face model access
   - Check internet connectivity during build

3. **Memory Issues**
   - Free tier has 512MB RAM limit
   - Consider upgrading for better performance

4. **GPU Issues**
   - Render doesn't support GPU on free tier
   - Model runs on CPU (slower but functional)

### Performance Optimization:

1. **Upgrade Plan**: Consider paid plans for better performance
2. **Model Optimization**: Use smaller models if possible
3. **Caching**: Implement result caching
4. **CDN**: Use CDN for static assets
